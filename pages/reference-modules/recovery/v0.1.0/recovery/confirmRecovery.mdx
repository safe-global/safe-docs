# `confirmRecovery` (0x064e2d0e)



Lets single guardian confirm the execution of the recovery request.
Can also trigger the start of the execution by passing true to '_execute' parameter.
Once triggered the recovery is pending for the recovery period before it can be finalised.




Defined in [` SocialRecoveryModule.sol`](https://github.com/safe-global/safe-modules/tree/recovery/v0.1.0/modules/recovery/contracts/generated-markups/candide-contracts/contracts/modules/social_recovery/SocialRecoveryModule.sol#Lundefined)

## Parameters

### `_wallet`

  - **Type:** `address`

  The target wallet.

  ```solidity focus=2
  (ISafe safe).confirmRecovery(
    0x...,
    [0x...],
    0,
    true
  );
  ```


### `_newOwners`

  - **Type:** `address[]`

  The new owners' addressess.

  ```solidity focus=3
  (ISafe safe).confirmRecovery(
    0x...,
    [0x...],
    0,
    true
  );
  ```


### `_newThreshold`

  - **Type:** `uint256`

  The new threshold for the safe.

  ```solidity focus=4
  (ISafe safe).confirmRecovery(
    0x...,
    [0x...],
    0,
    true
  );
  ```


### `_execute`

  - **Type:** `bool`

  Whether to auto-start execution of recovery.

  ```solidity focus=5
  (ISafe safe).confirmRecovery(
    0x...,
    [0x...],
    0,
    true
  );
  ```
