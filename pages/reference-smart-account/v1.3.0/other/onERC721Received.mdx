# `onERC721Received` (0x150b7a02)



Handle the receipt of an NFT

The ERC721 smart contract calls this function on the recipient
after a `transfer`. This function MAY throw to revert and reject the
transfer. Return of other than the magic value MUST result in the
transaction being reverted.
Note: the contract address is always the message sender.




Defined in [` ERC721TokenReceiver.sol`](https://github.com/safe-global/safe-smart-account/tree/v1.3.0/contracts/interfaces/ERC721TokenReceiver.sol#L18)

## Returns



  - **Type:** `bytes4`

  `bytes4(keccak256("onERC721Received(address,address,uint256,bytes)"))` unless throwing




## Parameters

### `_operator`

  - **Type:** `address`

  The address which called `safeTransferFrom` function

  ```solidity focus=2
  (ISafe safe).onERC721Received(
    0x...,
    "0x..."
  );
  ```


### `_data`

  - **Type:** `bytes`

  Additional data with no specified format
